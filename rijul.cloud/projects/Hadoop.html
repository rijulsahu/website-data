Bigdata Analytics using Hadoop


Rijul_Sahu

November, 2018


Implementation of Competitor's Product Change Dashboards using Hadoop / Big Data.

Applicable to all processes operating in any data domain and facing data volume, data velocity, or data variety troubles




Description

The client, a leading U.K.-based electronic components manufacturer and distributor operating in 32 countries, wanted to track its competitors’ product pricing to strategize its new and old product lines.

It faced challenges while processing its monthly data volume (6 billion+ records), and also required support for building effective and time-sensitive competitive intelligence dashboards for all global competitors.

It wanted to develop a holistic Business Intelligence (BI) solution for middle and executive management, and reduce the lead time for publishing reports, to analyze trends using its historical data along with price changing products and entire product categories.

It wanted to perform competitive analysis on KPIs like price change, stock status, inventory, trend and matched articles with prominent competitors in the global market. The client’s existing technical architecture was incapable of effectively addressing these limitations.


Challenges

The key challenges faced in this process were:

  1. Time-consuming and manual data processing.

  2. Difficulty in analyzing trends using historical data, along with changing prices and product categories.

  3. Difficulty in performing competitive analysis due to huge data size.

  4. Inability to match ETAs.


Solution

With the aim of reducing the end-to-end time spent on manual and time-consuming activities in the Operations team.

I designed and deployed 6 nodes Hadoop cluster for data ingestion and data processing with 20 cores horsepower, 150GB RAM, and 2TB storage.

Earlier, because of technology incompetency, it wasn’t feasible to process the data crawled for 6 months, this was solved by storing all data into Hive tables in an optimized row columnar format and using Apache Pig language for business logic and calculations.

This simplified extremely complex ETL (extract transform and load) processes by integrating data and logic from multiple systems using Hadoop.

The entire process was automated from the data extraction layer to the end-user consumption layer.

My counterpart BI team also deployed interactive Tableau dashboards, which use Hive-aggregated data sources for known questions with an ad-hoc ability to pose new questions.


Benefits

Efforts in designing and implementing this solution led to:

  1. Save $150K cost of data processing, storage and analytics.

  2. Reduced TAT by 83% in dashboard creation and data processing.

  3. Pricing Data Warehouse was set up for more than 12 million inventory items for about 43 global competitors per month.

  4. Hadoop data processing solution enabled processing competitor data for 6 months with an inventory-level data of approx. ~6 billion, which is otherwise time consuming with traditional relational database management system like SQL Server.

  5. Lead time for report generation and strategy was reduced from 3 weeks to 1 day for reports and dashboards.

  6. Enabled processing data for all global markets like DE, FR, GB, IT, FR, JP, CN, etc. at once in batches, which wasn’t feasible otherwise.

  7. Saved ~3 weeks of manual efforts per month.

  8. Improved process efficiency by 99%.

  9. Saved 2 FTEs [Full time employees by automation].

  10. Removed human dependence on file processing.
