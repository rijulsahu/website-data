Terraform Template for Deploying VPC, Subnets, EC2, RDS & More


Rijul_Sahu

August, 2020


Developed reusable Terraform templates to automate AWS infrastructure deployment and enable a CI/CD workflow for cloud resource provisioning. The solution standardized DevOps practices and reduced manual deployment overhead across multiple teams.




You can find source code here, https://github.com/rijulsahu/vpc-2subnets-ec2-rds

Description

The client, a leading European multinational online food-delivery company headquartered in Berlin, operated across 40+ countries with partnerships exceeding 500,000 restaurants. To support the deployment of their Market Mix Model (MMM) analytics solution across multiple geographies, the DevOps team required a fully automated infrastructure setup. Their existing resource deployment process was manual, time-intensive, and lacked consistency. The goal was to design a scalable Terraform-based CI/CD framework for automated, error-free AWS resource provisioning.


Key Challenges

  • Manual and time-consuming setup of AWS cloud resources

  • Heavy dependency on multiple external IT vendors

  • Absence of formal DevOps processes

  • Delays in meeting project delivery timelines


Solution

To streamline the infrastructure deployment workflow, I designed and implemented an AWS Terraform template that:

  • Creates a VPC with two subnets for network segmentation

  • Launches EC2 instances to host applications and services

  • Provisions RDS instances for persistent data storage

  • Integrates seamlessly within the CI/CD pipeline for automated provisioning

This infrastructure framework allowed multiple teams to deploy and manage resources independently with consistent configurations. The IT team later used this environment to host an interactive web interface running the MMM model on the newly created EC2 instances.


Key Benefits

  • Established an internal knowledge base, removing vendor dependency

  • Enabled a full CI/CD-based DevOps implementation

  • Eliminated manual intervention, improving delivery accuracy and speed

  • Reduced deployment time while ensuring consistent, reproducible infrastructure setups
